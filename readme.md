# 触萌 (TouchMoe) —— 3D角色模型展示App AI开发计划

## 1. 项目概述

**项目名称**：触萌 (TouchMoe)  
**项目类型**：手机App（Android/iOS）  
**核心功能**：  
- 展示一个高质量的3D二次元角色模型  
- 支持单指旋转模型、双指缩放  
- 点击模型不同部位时触发对应的动画和粒子特效  
- 提供截图分享、重置视角、背景音乐开关等辅助功能  

**目标**：将《崩坏三》角色展示模块的核心体验独立出来，打造一款轻量、可爱、高交互感的模型展示应用。

---

## 2. 技术选型

| 类别 | 技术选型 | 用途 |
|------|----------|------|
| 跨平台框架 | Flutter | 构建App壳层，处理UI、手势、通信 |
| 3D渲染引擎 | Unity (作为Library嵌入) | 渲染模型、播放动画、处理后期特效 |
| 模型格式 | FBX → AssetBundle | 优化加载与内存 |
| 通信机制 | MethodChannel (Flutter ↔ Unity) | 传递手势参数、触发动画、同步状态 |
| 动画系统 | Unity Mecanim | 控制模型的Idle动画与触摸反馈动画 |
| 渲染管线 | Universal Render Pipeline (URP) | 实现二次元风格的卡通渲染 |
| 性能优化 | LOD、纹理压缩、Draw Call合并 | 保证中低端设备流畅运行 |

---

## 3. 项目目录结构（仅说明）

- **flutter_app/**：Flutter宿主工程，包含UI、手势处理、与Unity通信的代码。
- **unity_project/**：Unity渲染工程，包含模型、着色器、动画、通信桥接脚本。
- **build_output/**：最终生成的APK/IPA文件存放目录。

## 4. AI开发流程（按阶段执行）

### 阶段一：环境搭建与项目初始化（2天）

**目标**：建立Flutter+Unity混合开发环境，完成基础通信测试。

**任务列表**：
1. 在本地安装Flutter SDK、Unity Hub（Unity 2021.3 LTS版本）。
2. 创建Flutter项目，包名使用 `com.touchmoe.app`。
3. 创建Unity项目，模板选择“3D (URP)”，项目名称 `TouchMoeUnity`。
4. 配置Unity项目的导出设置，使其可以作为Library嵌入Flutter：
   - 在Unity中，Build Settings → 勾选“Export as Android Library”（Android）或“Export as Xcode Framework”（iOS）。
5. 在Flutter项目中添加 `unity_embedding` 插件依赖，并配置原生层（Android/iOS）以加载Unity库。
6. 在Flutter中创建 `UnityView` 组件，将Unity视图嵌入到App页面中。
7. 实现最简单的双向通信测试：Flutter发送一个测试字符串给Unity，Unity收到后打印日志；Unity发送一个测试字符串给Flutter，Flutter弹出Toast。

**产出**：  
- 可运行的Flutter项目，能成功显示Unity的默认场景。  
- 通信测试通过。

---

### 阶段二：模型导入与渲染优化（5天）

**目标**：将角色模型导入Unity，实现高品质的二次元风格渲染。

**任务列表**：
1. **模型准备**：获取一个完整的二次元角色FBX模型（含骨骼、动画），放入Unity项目的 `Assets/Models/Character` 目录。
2. **材质与贴图设置**：为模型分配对应的贴图（漫反射、法线、高光等），创建材质球并调整基础参数。
3. **卡通着色器开发**：编写一个Toon Shader（或使用URP自带Shader Graph制作），实现：
   - 明暗分界（阶梯式光照）
   - 描边效果（基于顶点膨胀）
   - 高光区域可调
4. **光照与环境**：在场景中布置三光源（主光、补光、背光），调整强度与颜色，使模型立体感强且符合二次元审美。
5. **后期特效**：添加Post-process Volume，启用Bloom、Vignette、Color Grading，增强画面氛围。
6. **资源打包**：将模型、动画、材质等打包为AssetBundle，实现动态加载。
7. **内存管理**：在Unity中编写资源管理器，确保模型加载后能正确卸载，避免内存泄漏。

**产出**：  
- 模型在Unity编辑器中达到理想渲染效果。  
- 支持从AssetBundle动态加载模型。

---

### 阶段三：交互功能实现（4天）

**目标**：实现触摸旋转、缩放、点击反馈等核心交互。

**任务列表**：
1. **手势捕获（Flutter侧）**：
   - 在Flutter的 `GestureDetector` 中监听 `onPanUpdate`（单指拖动）、`onScaleUpdate`（双指缩放）、`onTapDown`（点击）。
   - 计算旋转角度（水平旋转角度 = 拖动位移X × 系数，垂直旋转角度 = 拖动位移Y × 系数，并限制垂直角度在 -30° 到 60° 之间）。
   - 计算缩放值（缩放系数 = 当前缩放比例 × 手势scale值，限制在0.8~2.0之间）。
   - 记录点击坐标（相对于Unity视图的比例坐标0~1）。

2. **参数传递（Flutter ↔ Unity）**：
   - 通过MethodChannel将旋转角度、缩放值、点击坐标实时发送给Unity。
   - 约定通信方法名：`updateRotation`、`updateScale`、`onTap`。

3. **相机控制（Unity侧）**：
   - 编写 `CameraController` 脚本，接收旋转和缩放参数，控制主相机的旋转角度和与模型的距离。
   - 相机旋转采用围绕模型中心点的轨道控制（类似Orbit Camera）。

4. **点击反馈（Unity侧）**：
   - 在 `ModelInteraction` 脚本中，根据Flutter传来的点击坐标，发射射线检测模型碰撞体。
   - 获取命中点所在的部位（例如头部、手部、身体等，可通过模型骨骼或碰撞体名称区分）。
   - 根据部位调用对应的动画（如摸头→开心动画，摸手→挥手动画等）。
   - 在点击位置生成粒子特效（如爱心、星星），播放短暂音效。

5. **动画管理（Unity侧）**：
   - 使用Animator Controller管理多个动画状态（Idle、TouchHead、TouchHand等）。
   - 实现动画混合过渡，保证触摸反馈动画播放完毕后自动回到Idle。

**产出**：  
- 用户可通过触摸旋转/缩放模型。  
- 点击模型不同部位，模型做出不同反应（动画+特效）。

### 阶段四：UI与用户体验（3天）

**目标**：设计可爱的UI界面，增加辅助功能，提升整体体验。

**任务列表**：
1. **UI设计**：
   - 设计启动页（Splash Screen），使用项目Logo与主题色（粉色/蓝色系）。
   - 设计底部工具栏：三个圆形按钮（截图、重置视角、音乐开关）。
   - 设计引导页（首次启动时，用几张图片说明如何旋转、缩放、点击互动）。
2. **截图分享功能**：
   - Flutter侧捕获当前屏幕图像（包含Unity视图和UI），保存到相册。
   - 添加分享功能，可调用系统分享面板发送截图。
3. **音效与音乐**：
   - 准备一首轻快的背景音乐（BGM），默认开启，提供静音切换。
   - 为点击交互准备短音效（如“叮”或“噗”），在Unity中通过点击事件触发播放。
4. **重置视角**：
   - 点击“重置”按钮时，Flutter向Unity发送重置指令，将相机旋转角度归零、缩放恢复为1.0。
5. **UI适配**：
   - 确保在不同屏幕比例（全面屏、刘海屏）下，按钮位置正常，Unity视图比例不变形。

**产出**：  
- 完整UI界面，符合“可爱”主题。  
- 截图分享、音乐开关、重置视角功能可用。

### 阶段五：性能优化与兼容性测试（3天）

**目标**：确保App在中低端设备上稳定运行，无崩溃、卡顿。

**任务列表**：
1. **性能监控**：
   - 在开发过程中使用Unity Profiler和Flutter DevTools监控FPS、CPU/GPU占用、内存。
   - 针对性能瓶颈进行优化。
2. **图形优化**：
   - 为模型生成LOD（Level of Detail）等级，距离相机远时使用低模。
   - 纹理压缩为ASTC（Android）或PVRTC（iOS），减少内存和带宽。
   - 合并Draw Calls，减少渲染批次。
3. **兼容性测试**：
   - 准备至少3款不同档次的Android真机（如骁龙6系、8系）和2款iPhone（如iPhone 8、iPhone 13）。
   - 测试安装、启动、模型加载、交互响应、截图、重置等功能。
   - 测试不同系统版本（Android 6~13，iOS 12~16）。
4. **问题修复**：
   - 记录并修复所有崩溃、黑屏、模型不显示、动画卡顿等Bug。
   - 优化内存占用，确保长时间运行不会导致内存泄漏。

**产出**：  
- 性能报告（FPS稳定在30以上，内存占用<200MB）。  
- 兼容性测试通过清单。

## 7. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| Flutter与Unity通信延迟 | 旋转/缩放不跟手，体验差 | 减少通信频率（仅在必要时同步），或在Unity中做平滑插值 |
| 模型渲染性能不足 | 低端机发热、掉帧 | 提前做LOD，提供画质选项（高/中/低） |
| 动画切换不流畅 | 交互生硬 | 使用动画过渡（CrossFade），避免频繁触发 |
| iOS打包签名问题 | 无法生成IPA | 提前准备Apple开发者账号，熟悉Xcode签名流程 |
| 内存泄漏 | 长时间运行后闪退 | 在Unity中使用Profiler排查，确保资源正确卸载 |
